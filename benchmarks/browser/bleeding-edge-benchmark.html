⚡ XPB Bleeding Edge Benchmarks (2025)

Testing the absolute limits of browser performance using Unsafe Accessors, Native Base64, and Shared Memory.

1. Native Base64 vs atob()

Comparing the new Uint8Array.fromBase64 against standard atob().

2. "No-Decode" Accessors (Zero-Copy)

Comparing JSON.parse style object creation vs Accessor pattern (on-demand read).

3. Shared Memory (Zero-Copy Workers)

Comparing postMessage (Copy) vs SharedArrayBuffer (Zero-Copy).

⚠️ Requires COOP/COEP headers to be set on the server.

4. String Array (Lazy View)

Comparing JSON.parse style full decode vs Lazy View (Scan-only).

5. BYOB Stream -> SharedArrayBuffer

Simulating 50MB network stream directly into Worker memory.

6. Wasm SIMD (ZigZag Decode)

Comparing Scalar JS vs Wasm Scalar vs Wasm SIMD (128-bit).

7. Large Message View (Lazy)

Comparing Standard Decode vs Lazy View for large objects.
