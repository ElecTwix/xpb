# Migration Guide

## 0.4.x → 0.5.0 (Go codegen defaults)

0.5.0 changes the **default** Go code generated by `xpbc`. The wire format is
**unchanged** — data encoded by 0.4.x decodes identically under 0.5.0 and vice
versa, and the cross-language conformance vectors are byte-identical. Only the
generated Go *struct shape* and *decode aliasing semantics* change. Generated
TypeScript, C, Java, Lua, and Rust are unaffected.

Two defaults flipped:

| Aspect | 0.4.x default | 0.5.0 default | Opt-out flag |
| --- | --- | --- | --- |
| Optional fields | `*T` pointer | value `T` + `HasX bool` | `--go-optional-style=pointer` |
| `bytes` decode | copy | zero-copy (aliases input) | `--go-safe-bytes` |

To regenerate 0.4.x-equivalent Go with no source changes, pass both opt-outs:

```sh
xpbc --lang=go --go-optional-style=pointer --go-safe-bytes schema.xpb
```

Otherwise, apply the changes below.

### 1. Optional fields: `*T` → value + `HasX`

An optional field `?string method` used to generate a pointer; it now generates
a value plus a presence bool. These changes fail at **compile time** (no silent
data bugs), so the compiler points you at every call site.

**Generated struct**

```go
// 0.4.x
type Msg struct {
    Method *string
}

// 0.5.0
type Msg struct {
    Method    string
    HasMethod bool
}
```

**Reading**

```go
// 0.4.x
if m.Method != nil {
    use(*m.Method)
}

// 0.5.0
if m.HasMethod {
    use(m.Method)
}
```

**Constructing**

```go
// 0.4.x
s := "market.subscribe"
m.Method = &s

// 0.5.0
m.Method = "market.subscribe"
m.HasMethod = true
```

Non-enum **message** optionals are unaffected — they stay `*T` (the pointer is
the natural absence marker), with no `HasX` bool.

### 2. `bytes` (and `string`) now alias the decode buffer

Decoding is now zero-copy by default: a decoded `[]byte` (and, as before,
`string`) **points into the buffer you passed to `Unmarshal`**. It is valid only
while that buffer is alive and unmodified.

This is safe for the common request-scoped pattern — decode, use, discard before
the buffer is reused. It is **not** safe if you retain decoded data past the
buffer's lifetime, e.g. reading successive messages into one reused/pooled
buffer:

```go
buf := make([]byte, 64*1024)
for {
    n, _ := conn.Read(buf)
    var m Msg
    _ = m.Unmarshal(buf[:n])
    cache[m.Id] = m.Payload // BUG under 0.5.0: aliases buf; next Read corrupts it
}
```

Two fixes — pick one:

**a. Clone what you keep**

```go
cache[m.Id] = append([]byte(nil), m.Payload...) // owns its bytes
```

**b. Opt out of zero-copy at generation time**

```sh
xpbc --lang=go --go-safe-bytes schema.xpb   # decoded bytes are copied, as in 0.4.x
```

Note: `string` fields have aliased the input buffer since before 0.5.0 — only
`bytes` changed in 0.5.0. If you reuse decode buffers, audit retained strings
too (`string(m.Field)` forces a copy).

### Performance

The new defaults are the fast path: generated decode/encode run with **zero
allocations** and no per-field pointer boxing or buffer copies (≈8.7 ns decode /
≈13 ns encode for a representative RPC message on an Apple M5, vs ≈1000 ns for
`encoding/json`). The opt-out flags trade that back for the 0.4.x ergonomics.
